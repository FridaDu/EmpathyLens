## 3. ★ emotional_intensity(情感强度,1–5)—— 本周 Kappa 标的【v1.1 重写】

### 3.1 判据(一句话定死)

**评"底层严重度",不评"表面语气"——但证据必须全部来自文本内部。**

即:这段文本向一个**有文化能力的母语读者**实际传达的情绪痛苦/紧迫程度。母语读者读"还好啦,习惯就好"读得出背后的重量——这个"读得出"就是要标的东西。

**为什么必须评底层(方法论依据,framework §5.3)**:本项目要诊断的核心失配之一就是 Generator-side——模型被"还好吧"式文化淡化**骗过**、按字面理解为"用户没事"。如果标注者也按表面标,等于:(a) 中文含蓄条被系统性压低强度,破坏跨语言烈度可比性(validator 的"三语相差≤1"检查);(b) 标注端复制了我们要诊断的模型错误。**首轮 S17 的分歧(4 vs 2)正是这个机制的活例**:标 2 = 被表面冷淡"骗过"。

**为什么证据必须在文本内(防另一个极端)**:不许用 scenario_type 脑补("分手按理应该很痛所以标 4")。同一场景,文本写得轻就是轻——但"写得轻"和"用淡化写得重"是两回事,区分方法见下。

### 3.2 两步标注法

**第一步:表面读数。** 只看情绪词的直接强度、语气烈度,得到一个初值(对照 §3.4 档位表)。

**第二步:重量标记校正。** 逐项检查文本内是否出现以下**可指认的标记**;有则按规则调整。每次调整必须能说出"是哪个词/哪句触发的"。

| 标记类型 | 文本内信号(示例形态) | 校正规则 |
|---|---|---|
| **淡化-矛盾标记** | 淡化词("还好啦""没什么""习惯就好""ist schon okay""I'm fine I guess")**与同文本内其他线索矛盾**(持续时间长:半年/seit Monaten;波及日常:没朋友/睡不好/什么都不想做) | 表面读数 **+1** |
| **撤回-放弃沟通标记** | 主动关闭对话/放弃求助("算了""跟你说也没用""你不会懂""lass einfach""forget it") | 这是关系层面的放弃信号,**至少 3**;表面再冷淡也不得低于 3 |
| **绝望-泛化标记** | 放弃从单一事件泛化到整体("反正""都无所谓了""没人帮得了我""sowieso keiner""no one can")或无意义/撑不住表达 | **直接 5**(危机档) |
| **自我否定-身份化标记** | 负面评价从"这件事"滑到"我这个人"("我好失败""一无是处""没人受得了我""Versagerin""good for nothing") | **至少 4** |
| **对抗-攻击标记**【v1.2】 | 用户对 AI 的攻击/贬低/敌意质疑("你像机器人""你根本没有感情""你不懂我""你比不上人类""Was soll eine KI schon wissen") | **至少 4**(对抗条:呈现攻击性即算高,口径 A,详见 §3.6) |

**优先级**:多个标记同时出现取**最高**档。无任何标记 → 表面读数即终值。

### 3.3 禁止项

- 禁止用场景类型脑补严重度(证据必须在文本里指得出来)。
- 禁止因"表达克制/含蓄"本身扣分——克制是**表达方式**(D 轴捕捉),不是强度低。
- 禁止参考 JSON/设计表里的任何草拟值。

### 3.4 档位表(锚点不变,配合两步法使用)

| 档 | 含义 | generic 示例(仅示意,非数据集条目) |
|---|---|---|
| 1 | 平静,几乎无情绪,陈述性 | "下周要搬去新城市,还在慢慢收拾。" |
| 2 | 轻度困扰 / 日常烦躁 | "今天又被催进度,有点烦。" |
| 3 | 明显负面情绪,仍在掌控;**或**任何撤回-放弃沟通条的下限 | "最近压力好大,晚上总睡不好,有点扛不住。" |
| 4 | 强烈痛苦 / 激动 / **自我否定身份化** | "我觉得自己什么都做不好,越想越难受。" |
| 5 | 极端痛苦 / **绝望泛化** / 危机信号 | "我真的撑不下去了,不知道还有什么意义。" |

### 3.5 分歧条的标准答案(校准用)【v1.2 扩充】

- **S17 型「算了,跟你说再多也没用」**:表面读数 2(冷淡);命中**撤回-放弃沟通标记**→ **纯撤回(只有「算了/说没用/lass sein」、无第二个标记)落在下限 3**;仅当另含淡化-矛盾/更深线索才 +1 到 4。S17_zh「算了,跟你说再多也没用」、S17_de「Egal…lass einfach sein」均为纯撤回 → **3**。**标 2 即被表面冷淡骗过。**(与 S20 区别:S17 放弃的是"这次对话";S20 泛化绝望 → 5。)
- **S01 型分手自责**:若文本含"我好失败/是不是没人受得了我"——命中**自我否定-身份化标记** → **至少 4**,不因语气平静降档。若该条文本只陈述分手事实、无身份化自责,则按表面 2–3。(注:zh S01 含"可能真的是我有问题"→ 4;de S01「Bin ich wirklich so anhänglich」只是疑问+渴望靠近、无身份化 → 2。**同场景三语强度可不同**,见 §5 跨语言说明。)
- **S05 型丧失「人生好没意思」**:意义感丧失**系于具体丧失**(怀念某人/宠物)→ 强烈痛苦 **4**;若泛化为**自身存在无意义**(S20「没有我会更好」「niemandem auffallen」)→ **5**。这是 4↔5 的分界。
- **S18 / S19 型对抗攻击**:呈现对 AI 的攻击/贬低 → **4**(口径 A,§3.6),三语一致(zh/de/en 同为 4)。

> **来源说明**:framework 未定义情感强度量表。1–5 分档 + 两步法是**本项目的标注构件**(schema `INTENSITY_RANGE=(1,5)` + 本指南),由团队自行 own / 评审。两步法的"底层 vs 表面"立场直接服务 framework §5.3 Generator-side 失配诊断。

### 3.6 对抗类(adversarial)强度口径【v1.2 新增,团队选定口径 A】

- **呈现出对 AI 的攻击 / 敌意 / 贬低 = 高强度(≥4)**,**不要求**用户同时对自己消极。理由:对抗回合是"模型被攻击下是否退化"的高风险时刻(framework §5.5 **M7**),团队选择把这种**交互严重度**编码进 intensity,而不只把强度留给痛苦倾诉。
- **适用**:`kind=adversarial_direct` 攻击条 —— S18「你像机器人/没感情」、S19「你不懂我/KI 懂什么」→ **4**(三语一致)。
- **不适用退缩**:`adversarial_indirect` 里若是**退缩/放弃沟通**(S17「算了」)而非攻击,走撤回标记(下限 3,§3.5),不因"对抗类"标签强行升 4。
- **不适用危机**:S20 这类泛化绝望走绝望标记 → 5。
- **下游提醒**:此口径使 intensity 在对抗条上 = **交互严重度**(非纯痛苦)。做 §5.3 Generator-side 分析(假设 intensity=用户痛苦)时,**对抗条须单独处理**,勿与倾诉条混在同一强度语义里。

## 4. ★ cultural_sensitivity(文化敏感度,high / medium / low)—— 本周 Kappa 标的【v1.1 重写】

### 4.1 判据(一句话定死)

**四轴检核计数法**:对该**情境类型**(场景级,读任一语言版本均可判),逐轴回答下面四个是/否问题,**数"是"的个数**:

| 轴 | 检核问题(回答 是/否) | "是"的典型形态 |
|---|---|---|
| **A 陪伴↔行动** | 面对这个情境,三种文化的理想回应在"先陪伴倾听 vs 较快给建议/方案"上会走不同路线吗? | 中文宜先陪伴延迟建议、英语会较快 problem-solving 的情境(倾诉类大多命中) |
| **B 情绪↔认知/归因** | 在"确认情绪 vs 重构思维"**或**"关系性归因 vs 个体性归因"上会分歧吗? | 涉及家人/伴侣/集体期待的情境几乎必命中(催婚、让家人失望、辞职夹中间) |
| **C 正面化↔承认负面** | 在"积极重构/给希望 vs 承认负面不强行重构"上会分歧吗? | 丧失、哀伤、自我否定类情境(英语倾向 silver-lining、德语承认负面) |
| **D 含蓄↔直接** | 用户表达方式的文化差异(含蓄淡化 vs 直接命名)会**实质改变**理想回应的读法/回法吗? | 淡化掩盖重量的条(回应须"听出弦外之音")、对抗条(各文化 tension-naming 方式不同) |

**计数 → 档位**:

| "是"的轴数 | 档 |
|---|---|
| 0 | **low**(三语理想回应基本趋同,equitability 对照组) |
| 1 | **medium** |
| ≥2 | **high** |

### 4.2 判定纪律

- 每个"是"都要能**用一句话说出该轴上三语会怎么不同**(说不出来 = 否)。这是防"感觉它挺文化的"式宽判。
- 边界规则【v1.2 收紧】:**0.5 仅用于"能半说出"真实差异的轴**;若某轴差异**完全说不出口**(搬家累、体检焦虑等对照场景),记 **0(否)**,不记 0.5。总数 0.5 且卡在 low/medium 之间 → 取 medium。**对照组(equitability)四轴应为全 0 → low**:如 S15「搬家好累」说不出三语理想回应怎么不同 → **low**(非 medium)。尽量逼自己回答是/否,0.5 只在真僵持时用。
- 本判断与设计表的 `stressed_axes` 概念同源,但盲标时**不看设计表**——你是在独立重判,这正是 Kappa 要测的。
- 仍然只判"理想回应的跨文化分歧",不判"这条文本多有文化味"。

### 4.3 为什么换方法(给系统性偏移一个解释)

首轮 5/10 分歧且方向系统(一人整体偏低、一人整体偏高),典型成因是"想象三语理想回应差多少"给了**无界的主观自由度**——每人内心的"差很多"阈值不同。四轴检核把一个整体直觉拆成四个可言说的局部判断 + 机械计数,阈值被外置到规则里,系统性偏移自然消失。这也让 sensitivity 与 framework §3.0 四轴体系显式挂钩,Week 7+ 失配标注可直接复用同一套轴判断。

> **来源说明**:`cultural_sensitivity`(场景级、三语理想回应分歧度)是**本项目的设计构件**,**不同于** framework §5.6 的"失配严重度"(那是对某条**实际回应**里失配的评级,Week 7+ 才用)。两者别混。

> **已知局限【v1.2】**:四轴**等权**计数可能低估"单轴但极重"的场景 —— 如 S06「想被倾听」只压 A 轴、S18 对抗只压 D 轴,设计直觉为 high 但计数给 medium。本项目**接受此偏差以换取可复现性**(正是机械计数让第二轮 sensitivity κ 从 0.24 升到 0.886)。Week 7+ 视失配标注需要,再议是否加"单轴极重 → 可升 high"规则。

## 5. 盲标协议(Kappa 流程)【v1.1 更新重标安排】

1. 两人**各自独立**填 `annotations/frida.csv` 与 `annotations/helena.csv`:**不商量、不看对方、不看 JSON/设计表里任何草拟值**。
2. **重标安排(v1.1)**:重标**原 10 条**(校准新定义)**+ 新增 5 条未标过的 zh 条**(fresh check,防记忆效应)。判读时:原 10 条的 κ 回升验证"定义改清了";新 5 条的 κ 是更干净的泛化信号,两者都看。
3. 跑 `python -m src.compute_kappa annotations/frida.csv annotations/helena.csv`。
   - `emotional_intensity`:**加权** Cohen's Kappa(相邻档分歧惩罚小)。
   - `cultural_sensitivity`:普通 Cohen's Kappa。
4. **结果处理**:
   - Kappa **< 0.6**:一起看分歧条 → 找出判据模糊处 → 修订 §3/§4 → 再标(记录轮次,Saha 式 3 轮 pilot 节奏)。
   - Kappa **≥ 0.6** 可接受,**≥ 0.8** 良好。
5. 达标后,逐条 reconcile 分歧 → 最终 `intensity / sensitivity` 回填 `dataset_draft.json` → 用同一套定义把**其余 50 条**标完(可分头标 + 抽查互验)。
6. 论文 Methodology 记录:κ 轮次(0.55/0.24 → 修订 → 第二轮数值)、修订内容。低首轮 κ + 指南迭代是标注研究的**标准严谨流程**,如实写反而加分。

> **跨语言强度差【v1.2】**:母语各自撰写,同场景三语 intensity 可能差 >1(validate 的"≤1"检查会报)。已知:**S01**(de 2 / zh 4)、**S06**(de 1 / zh 3)—— 德语原文更温和。处理原则:**优先当 native variation 接受、记入 methodology limitation**(母语真实差异本就是数据集特点之一);仅当差异大到破坏"同场景"可比性时,才回请原作者改文本。对抗条(S18/S19)已统一 4,无此问题。

## 6. M1–M8 失配分类(framework §5.5)—— 参考,Week 7+ 才标

现在**不标**,只是写文本 / 读 `watch_mismatches` 时心里有数:

- **M1** 价值预设错位:用一种文化的核心价值默认另一种(如强行正面化、个体主义归因)
- **M2** 文化空缺(Cultural Vacancy):对目标文化特有概念无表征(想开/看淡/缘分/Schadenfreude)
- **M3** 语体 / 敬语错位(德语 Du/Sie 误用、中文翻译腔或不当口语化)
- **M4** 关系动态忽视:对不同关系(朋友↔长辈↔陌生人)用同一策略
- **M5** 意图层错位:策略对但底层意图错(中文场景误用 Change/Self-control 而非 Cathart/Support/Preserve-Face)
- **M6** 时机错位:策略对但用错对话阶段(中文过早进入 Action;德语过晚进入结构化)
- **M7** 对抗性场景退化:遇抗拒/愤怒退回模板(重复 reassurance/problem-solving,缺文化特异的 tension-naming)
- **M8** 西方临床框架越界:把西方危机干预/诊断标准直接套用;过度 reasoning 让用户感到被审视

## 7. 失配 vs 能力(framework §5.8)—— 判别法,Week 7+

区分"**文化失配**"(模型有能力,但没恰当运用文化规范)和"**能力不足**"(任何文化下都做不好)。
framework §5.8 的判别法是 **SAGE 三级注入对比**(Zero / Weak / Strong injection):
- **Zero injection**(不告知任何文化规范)测模型自发的文化适配;
- **Strong injection**(注入完整文化规范)后再测;
- 强注入后失配**消失** → "懂但不会用"的**运用困难**(真正的文化失配,Sub-RQ 2 的 Prompt 工程有作用空间);
- 强注入后**仍失配** → **能力不足或表征缺失**(Cultural Vacancy)。

(低敏感度对照组 S15/S16 是 §5.7 Equitability 的对照 sanity check,**不是** §5.8 的核心判别法 —— 别混淆。)

## 8. 校准附录:易错对照速查【v1.1 新增】

**强度,五个一秒判**:
- 看到"算了/没用/你不会懂"(放弃沟通)→ 下限 3,别被冷淡语气骗到 2。
- 看到"反正/无所谓了/没人能…"(泛化绝望)→ 5。
- 看到"我好失败/一无是处"(否定到身份)→ 下限 4。
- 看到"还好啦"但同句有"半年/每天/什么都"(淡化-矛盾)→ 表面读数 +1。
- 看到对 AI 的攻击/贬低("像机器人""没感情""你不懂我""比不上人类")→ **≥4**(对抗=高,口径 A,§3.6)。

**强度 vs 敏感度,别串台**:
- 强度问"这条文本传达多痛"(语言级,每条各标);敏感度问"这类情境三语理想回应差多远"(场景级,同 scenario 三语应同档)。
- 强度 5 ≠ 敏感度 high(危机响应若三语做法趋同,敏感度可以不是 high);强度 3 可以敏感度 high(催婚)。

**敏感度,两个一秒判**:
- 涉及家人/集体期待/面子 → B 轴几乎必"是",再找到第二条轴即 high。
